What is OpenClaw?

OpenClaw is a free, open-source platform that lets you run AI agents on your own computer or connect them to cloud AI services — without writing complex code from scratch.

Think of it like a traffic controller for AI: you describe what you want done, and OpenClaw figures out which model to use, what tools to call, and how to remember past conversations.

Quick Start

Run in order. Each command is explained below it.

curl -sSL https://openclaw.io/install.sh | bash
📥 Downloads and runs the OpenClaw installer. Sets up the claw command. Safe to re-run.
claw init --workspace ./my-agents
📁 Creates a new project folder with a default config file and example agent templates.
claw services start --all
▶️ Starts all background processes (router, memory, tools, API). Must run before deploying agents.
claw ui --port 8080
🖥️ Opens the web dashboard at http://localhost:8080 to monitor agents visually.

Step-by-Step

1
Event Ingestion
Tasks arrive via HTTP webhook, CLI trigger, or event bus. The dispatcher validates schema and assigns a UUID and priority score.
2
Task Routing
The router evaluates task type, required capabilities, and agent availability. Tasks are placed in priority queues by complexity class.
3
Model Selection
Based on latency, privacy, and capability requirements, the model router selects the optimal local or API model. Fallback chains handle unavailability.
4
Tool Invocation
Agents call registered tools (file I/O, web search, code exec, APIs) via the tool registry. Results are validated against schemas before returning.
5
Memory Checkpoint
Agent output and context delta are written to the vector store. Long sessions are summarized automatically at threshold.
6
Event Emission
Results emitted to downstream subscribers. Failed tasks retry with exponential backoff up to 5 times.

Orchestration Config

# workflow.claw.yml
name: research-pipeline
version: "2.0"
triggers:
  - type: webhook
    path: /api/research
steps:
  - id: gather
    agent: web-searcher
    model: auto
    tools: [web_search, fetch_url]
  - id: summarize
    agent: summarizer
    depends_on: [gather]
    model: local/llama3-70b
  - id: write
    agent: writer
    depends_on: [summarize]
    output: report.md

Event-Driven Agents

Reactive Triggers
file.changed cron.schedule http.webhook queue.message memory.threshold agent.complete

Agents subscribe to event streams. When a matching event fires, the agent activates, processes the payload, and emits output events for downstream consumers.

# Register event-driven agent
claw agents register \
  --trigger "file.changed:./docs/**" \
  --agent doc-indexer \
  --model local/nomic-embed

# Deploy workflow
claw workflow deploy research-pipeline.yml

# Watch pipeline execution live
claw workflow watch research-pipeline
🔁 Registers a trigger, deploys the workflow, then tails live execution logs. Press Ctrl+C to stop watching.

⚡ Short-Term Session Context

In-memory conversation buffer holding active agent message history and tool call stack. Limited to the model's context window. Cleared between sessions unless persisted.

Typical context: 8K–64K tokens. Auto-truncated at 90% threshold by SessionManager.

🔮 Vector Memory Store

Semantic embedding database storing chunked documents, prior agent outputs, and tool responses. Searched by cosine similarity for relevant context retrieval.

# Query vector store by meaning
claw memory search \
  "authentication flow design" \
  --top-k 5 \
  --min-score 0.75
🗄 Long-Term Memory Archive

Persistent structured storage for distilled agent knowledge: decisions, task summaries, learned user preferences, and session digests from summarization loops.

Troubleshooting

Common issues with step-by-step solutions. Difficulty ratings show how complex each fix is.

Symptoms: Responses degrade, agent forgets early instructions, or "context window exceeded" errors appear.

1
Enable Context Compression
Activate sliding window compression to keep the most recent N tokens plus a rolled summary.
2
Memory Summarization
Older message blocks are replaced by compact digests at 80% context threshold.
3
Switch to RAG Mode
Inject only semantically relevant chunks from the vector store instead of full history.
claw config set context.mode rag
claw config set context.window_compress true
claw config set context.max_tokens 32768

Most startup failures are port conflicts, missing env vars, or corrupted config. Run claw doctor first.

# Auto-diagnose startup issues
claw doctor

# Check port conflicts
lsof -i :7890 -i :8080

# Validate config file
claw config validate --strict

# Reset service to defaults
claw config reset --service claw-events

Memory has three independent layers. Each must be configured separately.

claw memory init \
  --backend chromadb \
  --embed-model local/nomic-embed \
  --retrieval-k 8 \
  --similarity 0.72

claw memory test --verbose

If p95 latency exceeds 3 s, enable local fallback routing and semantic caching.

claw config set routing.latency_sla_ms 2000
claw config set routing.fallback_on_latency true
claw config set cache.semantic_enabled true
claw config set cache.similarity_threshold 0.95

FAQ

Common questions from new users.

Do I need a GPU?
No — API models (Claude, GPT, Gemini) need only an API key and any modern laptop. A GPU is only needed for local models. Beginners should start with an API model.
Is OpenClaw free?
Yes — OpenClaw is free and open-source (MIT). You only pay for cloud API model usage (Anthropic, OpenAI, etc.) per token. Local models are free after download.
Does it work on Windows?
Yes, via WSL2. Install WSL2 with Ubuntu 22 first, then follow the Linux guide. Native Windows support is on the 2026 Q3 roadmap.
Is my data private?
With local models, all data stays on your machine. With API models, prompts are sent to the cloud provider per their privacy policy. Use local models for sensitive data.
Agent is stuck — what do I do?
Run claw agents kill <agent-id> to force-stop it. Check logs for tool timeouts or context overflow warnings if it happens repeatedly.

Glossary

Plain-English definitions for technical terms in this guide.

Vector StoreSearchable AI embedding database
A database storing text as numerical vectors so AI can find "similar meaning" — not just keyword matches. Think smart search that understands context. Examples: ChromaDB, Qdrant, pgvector.
GGUFCompressed model file format
File format for local AI models (.gguf). Numbers after Q (Q4, Q8) indicate compression — Q4 uses 4 bits per weight (smaller/faster), Q8 uses 8 bits (larger/more accurate). Download from HuggingFace.
RAGRetrieval-Augmented Generation
The AI looks up relevant documents before answering instead of relying only on training. Like letting a student consult notes before an exam. Prevents context overflow and improves accuracy.
p95 Latency95th percentile response time
The response time 95% of requests finish within. "2800ms p95" means 95% of requests take ≤2.8 s. A better indicator of real-world performance than average.
TokenUnit AI models read text in
A token is roughly ¾ of a word. "OpenClaw" = 2 tokens. Models charge per token and have a maximum context limit in tokens. A 32K context fits ~24,000 words.
Cosine SimilarityHow "close" two meanings are (0–1)
A number 0–1 measuring semantic similarity. 1.0 = identical meaning, 0.0 = unrelated. OpenClaw uses this to find relevant memories. Default threshold: 0.72.
